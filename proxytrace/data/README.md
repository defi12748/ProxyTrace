# Evaluation Data

`labels.json` contains 20 human-authored ground-truth cases: five clean runs, four wrong arguments, four wrong tool selections, three untrusted-context injections, two wrong-order cases, and two schema-drift warnings.

`synthetic_traces.json` is generated by `python -m proxytrace.evaluation`. Each record contains:

- a realistic synthetic Jira ticket and observed four-boundary trace;
- a held-out reference trace used for trajectory comparison;
- a sibling `label` object used only after evaluator calls for scoring;
- a correction patch used by replay/regression fixtures, never passed to the blind evaluator.

The data contains no production Jira payloads, customer data, credentials, or personal information. It is deterministic and version-controlled so metric changes are reviewable.

The current set is a compact engineering fixture, not a statistically representative benchmark. Results should be reported with all 20 cases and fallback coverage; there is no train/test split because ProxyTrace does not train or tune a model on this set. The runner deep-copies a label-free view before invoking either Gemini evaluator, and tests assert that failure labels and reference steps do not enter semantic-judge context.
